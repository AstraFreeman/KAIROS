## 1. Примітка на майбутнє: система перевірки завдань

Завдання на стрічці (`data/posts/*.json`) наразі містять лише текстовий контент без механіки перевірки відповідей. Щоб платформа стала повноцінно навчальною, потрібна автоматична валідація.

### Що потрібно:
- **Модуль `js/tasks.js`** — IIFE `KAIROS.tasks` з логікою перевірки різних типів завдань
- **Структуровані `task` об'єкти** в полі `content` кожного поста:
  ```json
  {
    "content": {
      "text": "Опис завдання...",
      "task": {
        "type": "quiz",
        "question": "Коли впала Західна Римська імперія?",
        "options": ["395 р.", "476 р.", "1453 р.", "1204 р."],
        "correct": 1
      }
    }
  }
  ```
- **Типи завдань**: quiz (вибір з варіантів), chronology (drag-and-drop впорядкування), source-analysis (відкрита відповідь з ключовими словами), true-false

### Інтеграція:
- `tracker.js` вже відстежує `completions` — потрібно додати результати перевірки
- `ui.js` потребуватиме нових компонентів для кожного типу задач
- `achievements.js` вже реагує на дії трекера — нові досягнення за правильні відповіді

## 2. Концепція зони «Форум Каїр» — простір рефлексії

### Ідея

Замість формату бару (занадто неформально для університету) — **інтелектуальний хаб**, стилізований під відкриту площу для дискусій. Назва «Каїр» (Каїрос → Каїр) створює асоціацію з Єгиптом та грецькою філософією одночасно.

### Навігація
- **Маршрут SPA**: `#cairo`
- **Сайдбар**: іконка колони або відкритої книги з підписом **«Каїр: Простір рефлексії»** у секції «Навігація» (поруч зі Стрічкою, Профілем, Досягненнями)

### Візуальна зміна
При переході на `#cairo` фон контентної зони змінюється на спокійніший — приглушений темніший відтінок (наприклад, `#2c2c3a` з світлим текстом або `#f0ece4` теплий) замість стандартного `#e9eaed`. Стиль реалізується через клас `.cairo-zone` на контейнері `#app-content`.

### Привітання
> «Залиште списи та сувої біля входу. Тут час зупиняється для роздумів.»

### Контент
JSON-файл `data/cairo.json` з масивом об'єктів:
```json
{
  "items": [
    {
      "id": "q1",
      "type": "quote",
      "text": "Хто не знає свого минулого, не вартий свого майбутнього.",
      "author": "Максим Рильський",
      "period": "suchasnist"
    },
    {
      "id": "f1",
      "type": "fact",
      "text": "У Стародавньому Єгипті кішки вважалися священними. За вбивство кішки карали смертю.",
      "period": "starodavnij-svit"
    },
    {
      "id": "a1",
      "type": "anecdote",
      "text": "Наполеон, побачивши єгипетські піраміди: 'Солдати! Сорок століть дивляться на вас з висоти цих пірамід!' Солдат з-за лав: 'І жодне не допоможе нести ранець...'",
      "period": "novyj-chas"
    }
  ]
}
```

### Механіка
- При кожному вході — **випадкова цитата/факт/анекдот**
- Кнопка **«Ще одна»** для показу наступного
- Можливо: фільтр за періодом або типом контенту

### Технічна реалізація

| Файл | Зміни |
|------|-------|
| `js/cairo.js` (новий) | IIFE модуль `KAIROS.cairo` з `render(param, container)` |
| `js/app.js` | Додати `KAIROS.router.register('cairo', ...)` + sidebar link |
| `data/cairo.json` (новий) | Масив цитат, фактів, анекдотів |
| `css/kairos.css` | Стилі `.cairo-zone` — інший фон, типографіка, анімація появи |
| `platform.html` | Додати `<script src="js/cairo.js"></script>` перед `app.js` |

### Скрипт завантаження (порядок у platform.html)
```
utils.js → tracker.js → ui.js → achievements.js → feed.js → profile.js → cairo.js → router.js → app.js
```
